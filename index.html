Homelab

Outside of work I run a three-node homelab on tiny PCs that I treat like a production environment. It’s where I prototype ideas in infrastructure, Kubernetes, GitOps, and platform engineering before bringing them into my day-to-day work.

  • Cluster & virtualization: three Proxmox nodes running a mix of VMs and LXC containers. A dedicated pfSense VM acts as the perimeter firewall. The plan is to manage the Proxmox layer itself with OpenTofu (Terraform) so the whole lab can be rebuilt from code.
  • Kubernetes layer: a highly available k3s cluster with one node scheduled on each Proxmox host (via LXC), so I can survive single-node failures without losing workloads.
  • GitOps & operations: the k3s cluster is managed with FluxCD and a GitOps repository where I define cluster topology, base services, applications, and policies declaratively. Secrets and sensitive values are encrypted; changes are rolled out automatically from Git, not from my laptop.
  • Networking & security: access to the homelab is only possible over OpenVPN and a strict local ruleset enforced by pfSense. I regularly test failure scenarios (node loss, misconfigurations) to keep the setup resilient and recoverable.

This setup gives me a permanent sandbox for HA design, observability, incident handling, and secure networking on real hardware—not just in the cloud console—and keeps my platform skills sharp beyond what I do 9–5.
